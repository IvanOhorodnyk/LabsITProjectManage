# Labs IT Project Manage - MVP

[![CI/CD Pipeline](https://github.com/MykolaMelnykSHI/LabsITProjectManage/actions/workflows/ci-cd.yml/badge.svg)](https://github.com/MykolaMelnykSHI/LabsITProjectManage/actions/workflows/ci-cd.yml)

**Production URL (Vercel):** [https://labs-it-project-manage.vercel.app/](https://labs-it-project-manage.vercel.app/)

## Опис ідеї MVP
Цей проєкт створений у рамках лабораторної роботи з управління ІТ-проєктами. Його мета — продемонструвати базові навички роботи з Git: ініціалізацію проєкту, налаштування стратегії розгалуження (Feature Branch/GitFlow), створення Pull Requests, проведення Code Review та вирішення конфліктів злиття. 

Цей репозиторій є спрощеним варіантом (Minimum Viable Product), який демонструє структуру взаємодії між членами команди розробки.

## Інструкція з запуску
Оскільки на даному етапі реалізовано лише структуру (layout) та базові конфігураційні файли, запуск проєкту як такого не вимагає специфічних інструментів, окрім браузера для перегляду статичних файлів `frontend/layout/index.html`.

Для перевірки роботи:
1. Склонуйте репозиторій:
   ```bash
   git clone <url_вашого_репозиторію>
   ```
2. Відкрийте файл `frontend/layout/index.html` у браузері.
3. Дослідіть історію комітів та гілок за допомогою команд `git log --oneline --graph --all` або через інтерфейс GitHub (Network graph).
